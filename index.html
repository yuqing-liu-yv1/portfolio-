Design & Accessibility Reflection

This single-page template prioritizes clarity and update-ability. The information architecture follows a journalism-first order: About → Selected Work → Data/Projects → Resume → Contact. Cards in “Selected Work” present headline, a short deck, role, outlet/date, tags, and a link, enabling consistent scanning across media types (writing, photo/video, audio). Bootstrap provides a responsive grid and accessible components; custom CSS tightens spacing and improves color contrast beyond defaults.

Accessibility choices include semantic landmarks (header, nav, main, section, article, figure, footer), a visible “Skip to main content” link, descriptive alt text for the headshot, high-contrast text, focus outlines that persist for keyboard users, reduced-motion respect via prefers-reduced-motion, and descriptive links (no “click here”). Headings are hierarchical with a single h1.

Generative AI was used to draft the template structure, initial microcopy, CSS refactors, and accessibility checks. I reviewed and edited all content, replaced/verified links, and will continue to update clips. An AI usage log (ai.doc) documents prompts/tools and what I kept or changed. Future improvements: swap placeholder project cards with live embeds (Flourish/Datawrapper), add image galleries for photography, integrate a real email handler, and expand tags/filters when clips grow.
